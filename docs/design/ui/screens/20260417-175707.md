## 1. Overview

### 1.1 Screen Purpose

The TaskScreen displays a list of action items and tasks assigned to the client by their portfolio manager or required by the investment company. These tasks include document submissions, agreement signatures, information updates, and other compliance-related activities.

### 1.2 User Goals

| User Goal | Description |
|-----------|-------------|
| **View pending tasks** | See all tasks requiring action |
| **Understand task priority** | Identify urgent vs. non-urgent tasks |
| **Complete tasks** | Mark tasks as complete or take action |
| **View task details** | Access detailed information about each task |
| **Track progress** | Monitor overall task completion status |

### 1.3 Business Context

- Tasks are created by portfolio managers or compliance systems
- Tasks have due dates and priority levels
- Some tasks require in-app actions, others redirect to external systems
- Task completion affects account compliance status

### 7.3 Computed Values

```typescript
// Filtered tasks based on active filter
const filteredTasks = useMemo(() => {
  switch (filter) {
    case 'pending':
      return tasks.filter(t => t.status !== 'completed');
    case 'completed':
      return tasks.filter(t => t.status === 'completed');
    default:
      return tasks;
  }
}, [tasks, filter]);

// Sort order: Overdue > High > Medium > Low > Completed
const sortedTasks = useMemo(() => {
  return [...filteredTasks].sort((a, b) => {
    // Completed tasks go to bottom
    if (a.status === 'completed' && b.status !== 'completed') return 1;
    if (b.status === 'completed' && a.status !== 'completed') return -1;
    
    // Overdue first
    if (a.isOverdue && !b.isOverdue) return -1;
    if (b.isOverdue && !a.isOverdue) return 1;
    
    // Then by priority
    const priorityOrder = { high: 0, medium: 1, low: 2 };
    return priorityOrder[a.priority] - priorityOrder[b.priority];
  });
}, [filteredTasks]);
```
