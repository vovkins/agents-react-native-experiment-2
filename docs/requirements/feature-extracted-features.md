### FR-001: Аутентификация и авторизация

**Priority:** MUST HAVE

**Description:**
Система аутентификации и авторизации пользователей, обеспечивающая безопасный доступ к приложению. Включает регистрацию, вход, восстановление пароля и автоматический выход.

**Sub-features:**
1. **Регистрация через приложение** - Создание учетной записи после заключения договора с управляющей компанией
2. **Вход в систему** - Аутентификация по логину и паролю
3. **Восстановление пароля** - Сброс и изменение пароля через email/SMS
4. **Автоматический выход** - Завершение сессии после периода неактивности

**Functional Requirements:**
- FR-001.1: Форма регистрации с валидацией данных
- FR-001.2: Форма входа с полями логин/пароль
- FR-001.3: Функция "Забыли пароль?" с отправкой кода восстановления
- FR-001.4: Таймаут сессии (значение требует уточнения)
- FR-001.5: Безопасное хранение учетных данных на устройстве
- FR-001.6: Валидация формата email/телефона

**Edge Cases:**
- Неверный логин или пароль (отображение ошибки)
- Учетная запись не найдена при восстановлении пароля
- Истечение срока действия кода восстановления
- Слабый пароль при регистрации
- Уже существующий email/телефон при регистрации
- Потеря сети во время аутентификации
- Блокировка учетной записи после множества неудачных попыток

**Error Scenarios:**
- E001: Сервер аутентификации недоступен
- E002: Истечение срока токена сессии
- E003: Неверный формат введенных данных
- E004: Сетевая ошибка при запросе
- E005: Код восстановления истек или неверен

**Technical Constraints:**
- Шифрование данных при передаче (HTTPS/TLS)
- Безопасное хранение токенов на устройстве (Keychain iOS / Keystore Android)
- Соответствие требованиям регуляторов (требует уточнения)

**Dependencies:**
- Зависит от: API аутентификации бэкенда
- Блокирует: Все остальные функции (требуется аутентификация)

**Integration Points:**
- REST API бэкенда (endpoints: /auth/register, /auth/login, /auth/reset-password)
- Сервис аутентификации

### FR-002: Витрина продуктов

**Priority:** MUST HAVE

**Description:**
Раздел приложения для просмотра доступных инвестиционных стратегий доверительного управления и продуктов индивидуального управления. Отображает ключевые характеристики для принятия инвестиционных решений.

**Sub-features:**
1. **Список стратегий** - Каталог доступных стратегий доверительного управления
2. **Детальная информация стратегии** - Расширенное описание отдельной стратегии
3. **Индивидуальное доверительное управление** - Информация о продукте ИДУ
4. **Ключевые характеристики** - Отображение доходности, риска, минимальной суммы

**Functional Requirements:**
- FR-002.1: Отображение списка продуктов с фильтрами и сортировкой
- FR-002.2: Карточка продукта с ключевыми метриками
- FR-002.3: Детальная страница продукта
- FR-002.4: Отображение доходности (историческая, прогнозная)
- FR-002.5: Индикатор уровня риска
- FR-002.6: Минимальная сумма инвестиций
- FR-002.7: Описание стратегии инвестирования

**Edge Cases:**
- Пустой список продуктов (нет доступных стратегий)
- Продукт временно недоступен для инвестирования
- Некорректные или отсутствующие данные о доходности
- Исторические данные за недостаточный период
- Валютные расхождения в отображении сумм

**Error Scenarios:**
- E001: Ошибка загрузки списка продуктов
- E002: Продукт не найден при переходе по deeplink
- E003: Устаревшие данные в кэше
- E004: Несоответствие формата данных от API

**Technical Constraints:**
- Поддержка офлайн-режима (кэширование данных)
- Время загрузки не более 2 секунд
- Pull-to-refresh для обновления данных

**Dependencies:**
- Зависит от: FR-001 (Аутентификация)
- Независимо от других функций

**Integration Points:**
- REST API бэкенда (endpoints: /products, /products/{id})
- Система управления активами

### FR-003: Портфель активов

**Priority:** MUST HAVE

**Description:**
Отображение текущего портфеля активов клиента под управлением, включая структуру портфеля, текущую стоимость и историю изменений.

**Sub-features:**
1. **Текущий портфель** - Отображение состава портфеля активов
2. **Структура портфеля** - Визуализация распределения по активам/классам
3. **Текущая стоимость** - Расчет текущей рыночной стоимости портфеля
4. **История изменений** - Журнал операций и изменений портфеля

**Functional Requirements:**
- FR-003.1: Сводка портфеля (общая стоимость, изменение за период)
- FR-003.2: Список активов с долями и стоимостью
- FR-003.3: Визуализация структуры (круговая диаграмма)
- FR-003.4: Детали по каждому активу
- FR-003.5: История операций (покупки, продажи, начисления)
- FR-003.6: Фильтрация истории по дате и типу операции
- FR-003.7: Обновление данных в реальном времени или по запросу

**Edge Cases:**
- Портфель пуст (нет активов под управлением)
- Данные о стоимости устарели (рынок закрыт)
- Мультивалютный портфель (отображение в базовой валюте)
- Актив с нулевой стоимостью (ликвидация)
- Отрицательная доходность портфеля
- Частичное отображение данных (технические проблемы API)

**Error Scenarios:**
- E001: Ошибка загрузки данных портфеля
- E002: Нет данных для отображения истории
- E003: Несовпадение суммы по активам с общей стоимостью
- E004: Исторические данные недоступны

**Technical Constraints:**
- Поддержка офлайн-режима (кэширование последнего состояния)
- Время загрузки не более 2 секунд
- Время отклика не более 500 мс
- Обновление данных не реже чем раз в X минут (требует уточнения)

**Dependencies:**
- Зависит от: FR-001 (Аутентификация)
- Связано с: FR-002 (Витрина продуктов) - общие данные о продуктах

**Integration Points:**
- REST API бэкенда (endpoints: /portfolio, /portfolio/history, /portfolio/structure)
- Система управления активами

### FR-004: Чат с персональным менеджером

**Priority:** MUST HAVE

**Description:**
In-app чат для коммуникации клиента с персональным менеджером управляющей компании. Обеспечивает текстовую переписку, историю сообщений и уведомления.

**Sub-features:**
1. **Отправка сообщений** - Функция отправки текстовых сообщений менеджеру
2. **Получение сообщений** - Отображение входящих сообщений от менеджера
3. **История переписки** - Просмотр архива сообщений
4. **Уведомления** - Push-уведомления о новых сообщениях

**Functional Requirements:**
- FR-004.1: Интерфейс чата с историей сообщений
- FR-004.2: Отправка текстовых сообщений
- FR-004.3: Отображение статуса доставки сообщения
- FR-004.4: Отображение времени сообщения
- FR-004.5: Загрузка истории сообщений (пагинация)
- FR-004.6: Поиск по истории сообщений (опционально)
- FR-004.7: Уведомления о новых сообщениях
- FR-004.8: Счетчик непрочитанных сообщений

**Edge Cases:**
- Пустая история переписки (новый пользователь)
- Менеджер недоступен (нерабочее время, отпуск)
- Очень длинная история сообщений
- Потеря сети при отправке сообщения
- Отправка пустого сообщения
- Повторная отправка при ошибке
- Конфликт часовых поясов
- Ограничение длины сообщения (если есть)

**Error Scenarios:**
- E001: Ошибка отправки сообщения (сеть)
- E002: Сервер чата недоступен
- E003: История сообщений недоступна
- E004: WebSocket соединение разорвано
- E005: Сообщение не доставлено (таймаут)
- E006: История сообщений повреждена

**Technical Constraints:**
- Механизм real-time обновлений (WebSocket или polling) - требует уточнения
- Офлайн-очередь сообщений
- Retry-механизмы при ошибках сети
- Поддержка медиа-файлов (требует уточнения)
- Глубина хранения истории (требует уточнения)

**Dependencies:**
- Зависит от: FR-001 (Аутентификация)
- Связано с: FR-005 (Push-уведомления)
- Связано с: FR-007 (Профиль) - назначенный менеджер

**Integration Points:**
- REST API бэкенда (endpoints: /chat/messages, /chat/history)
- WebSocket сервер (real-time) или polling механизм
- Push-уведомления (FCM/APNs)

### FR-005: Push-уведомления

**Priority:** SHOULD HAVE

**Description:**
Система push-уведомлений для информирования пользователей о важных событиях: новых сообщениях, изменениях в портфеле, новых продуктах.

**Sub-features:**
1. **Уведомления о сообщениях чата** - Alert о новых сообщениях от менеджера
2. **Уведомления об изменениях портфеля** - Информирование о значимых изменениях
3. **Уведомления о новых продуктах** - Анонс новых инвестиционных возможностей

**Functional Requirements:**
- FR-005.1: Запрос разрешения на отправку уведомлений
- FR-005.2: Регистрация устройства для push-уведомлений
- FR-005.3: Обработка входящего уведомления
- FR-005.4: Навигация по tap на уведомление
- FR-005.5: Настройки типов уведомлений (включение/отключение)
- FR-005.6: Отображение badge с количеством непрочитанных

**Edge Cases:**
- Пользователь отказал в разрешении на уведомления
- Уведомление получено в фоне
- Уведомление получено при открытом приложении
- Множественные уведомления
- Устаревшее уведомление (событие уже обработано)
- Пользователь отключил уведомления в настройках

**Error Scenarios:**
- E001: Ошибка регистрации устройства в FCM/APNs
- E002: Push-сервис недоступен
- E003: Истек токен устройства
- E004: Уведомление не доставлено

**Technical Constraints:**
- Firebase Cloud Messaging для Android
- Apple Push Notifications Service для iOS
- Требуется настройка сертификатов и ключей

**Dependencies:**
- Зависит от: FR-001 (Аутентификация)
- Используется в: FR-004 (Чат)

**Integration Points:**
- Firebase Cloud Messaging (FCM)
- Apple Push Notifications Service (APNs)
- REST API бэкенда (endpoint: /notifications/register)

### FR-006: Биометрическая аутентификация

**Priority:** SHOULD HAVE

**Description:**
Альтернативный метод входа в приложение с использованием биометрических данных пользователя (отпечаток пальца, Face ID).

**Sub-features:**
1. **Touch ID / Face ID (iOS)** - Вход с биометрией на устройствах Apple
2. **Fingerprint / Face Unlock (Android)** - Вход с биометрией на устройствах Android

**Functional Requirements:**
- FR-006.1: Определение доступности биометрии на устройстве
- FR-006.2: Предложение включить биометрию после первого входа
- FR-006.3: Аутентификация с биометрией
- FR-006.4: Fallback на пароль при ошибке биометрии
- FR-006.5: Настройка включения/отключения биометрии
- FR-006.6: Отображение статуса биометрии в настройках

**Edge Cases:**
- Биометрия недоступна на устройстве
- Пользователь отказал в использовании биометрии
- Множественные неудачные попытки биометрии
- Изменение биометрических данных на устройстве
- Биометрия отключена после обновления ОС

**Error Scenarios:**
- E001: Биометрический датчик недоступен
- E002: Ошибка сканирования
- E003: Биометрия не распознана
- E004: Биометрия заблокирована (много попыток)

**Technical Constraints:**
- Требует наличия биометрического датчика на устройстве
- Зависит от OS-specific API
- Безопасное хранение биометрического ключа

**Dependencies:**
- Зависит от: FR-001 (Аутентификация)
- Связано с: FR-007 (Профиль - настройки)

**Integration Points:**
- iOS LocalAuthentication framework
- Android BiometricPrompt API
