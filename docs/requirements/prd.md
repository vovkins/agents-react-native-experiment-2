# Product Requirements Document (PRD)

## 1. Обзор продукта

### 1.1 Название продукта
Мобильное приложение для клиентов управляющей компании (название TBD)

### 1.2 Описание
Мобильное приложение на базе React Native для платформ iOS и Android, предназначенное для клиентов управляющей компании. Приложение обеспечивает прозрачность управления активами и прямую коммуникацию между клиентами и их персональными менеджерами.

### 1.3 Проблема
Состоятельные инвесторы, передающие активы в доверительное управление, испытывают трудности с:
- Отсутствием прозрачности в отношении доступных инвестиционных стратегий
- Недостаточным доступом к информации о текущем состоянии портфеля
- Сложностью коммуникации с персональным менеджером (телефон, email)

### 1.4 Решение
Мобильное приложение, которое предоставляет:
- Прозрачный доступ к витрине инвестиционных продуктов
- Реальное время отображения портфеля активов
- Удобный чат с персональным менеджером

### 1.5 Цели продукта
1. **Повышение прозрачности** - клиенты в любой момент видят состояние своих активов
2. **Улучшение коммуникации** - быстрый и удобный контакт с менеджером
3. **Увеличение вовлеченности** - клиенты активнее взаимодействуют с компанией
4. **Конкурентное преимущество** - современный цифровой канал взаимодействия

## 2. Целевая аудитория

### 2.1 Основная персона: "Александр - Инвестор"

**Профиль:**
- **Возраст:** 35-55 лет
- **Доход:** Высокий (топ-менеджеры, предприниматели, наследники)
- **Образование:** Высшее, часто финансовое или MBA
- **Активы:** От $500K в доверительном управлении

**Характеристики:**
- Активно использует смартфон (iPhone или Android)
- Ценит время и удобство
- Требователен к качеству сервиса
- Интересуется инвестициями, но не углубляется в детали
- Предпочитает персонализированный подход

**Боли:**
- "Я хочу видеть, как работают мои деньги, но не хочу звонить менеджеру"
- "Мне неудобно искать информацию в email-отчетах"
- "Хочу быстро задать вопрос, но не всегда дозваниваюсь"

**Потребности:**
- Быстрый доступ к информации о портфеле
- Понимание доступных инвестиционных стратегий
- Прямая линия связи с менеджером

### 2.2 Вторичная персона: "Елена - Бизнес-леди"

**Профиль:**
- **Возраст:** 40-50 лет
- **Статус:** Владелец бизнеса
- **Активы:** Диверсифицированные ($200K-1M в ДУ)

**Характеристики:**
- Очень занята, мало свободного времени
- Пользуется приложением рано утром или поздно вечером
- Нуждается в кратких и структурированных данных
- Ценит безопасность и конфиденциальность

## 3. Функциональные требования

### 3.1 Приоритет P0 - Must Have (Обязательно для MVP)

#### FR-001: Аутентификация пользователя
**Описание:** Безопасный вход в приложение с использованием email/пароля
**Функции:**
- Вход с email и паролем
- Автоматический выход после 30 минут неактивности
- Восстановление пароля через email

**Критерии приемки:**
- Пользователь может войти в систему
- Сессия автоматически завершается через 30 минут бездействия
- Пользователь может восстановить забытый пароль

#### FR-002: Экран витрины продуктов
**Описание:** Просмотр доступных инвестиционных стратегий и продуктов ДУ
**Функции:**
- Список стратегий доверительного управления
- Детальная информация по каждой стратегии (описание, доходность, риск, минимальная сумма)
- Карточка продукта индивидуального доверительного управления (ИДУ)
- Фильтрация по типу стратегии, уровню риска

**Данные для отображения:**
- Название стратегии
- Описание инвестиционного подхода
- Историческая доходность (график/цифры)
- Уровень риска
- Минимальная сумма инвестиций
- Управляющий/команда

**Критерии приемки:**
- Пользователь видит все доступные стратегии
- Можно открыть детальную карточку каждой стратегии
- Отображается актуальная информация о доходности

#### FR-003: Экран портфеля клиента
**Описание:** Просмотр текущего портфеля активов под управлением
**Функции:**
- Общая стоимость портфеля
- Структура портфеля (круговая диаграмма по классам активов)
- Список позиций с текущей стоимостью
- Изменение стоимости за период (день, неделя, месяц, год)
- График динамики стоимости портфеля

**Данные для отображения:**
- Общая стоимость портфеля в валюте
- Изменение в % и абсолютном выражении
- Структура: акции, облигации, фонды, кэш, прочее
- Топ-10 позиций
- Дата последнего обновления

**Критерии приемки:**
- Пользователь видит актуальную стоимость портфеля
- Можно увидеть структуру портфеля
- Отображается динамика изменений

#### FR-004: In-app чат с менеджером
**Описание:** Прямая коммуникация с персональным менеджером
**Функции:**
- Список предыдущих сообщений (история)
- Отправка текстовых сообщений
- Получение уведомлений о новых сообщениях
- Отметки времени отправки/получения
- Статус прочитано/не прочитано

**Критерии приемки:**
- Пользователь может отправить сообщение менеджеру
- История сообщений сохраняется
- Уведомления приходят вовремя (в течение минуты)

#### FR-005: Профиль пользователя
**Описание:** Просмотр и редактирование профиля
**Функции:**
- Отображение личных данных клиента
- Контактная информация
- Информация о менеджере
- Настройки уведомлений
- Выход из системы

## 4. Нефункциональные требования

### 4.1 Производительность
- **NFR-001:** Время запуска приложения ≤ 3 секунды
- **NFR-002:** Время загрузки экранов ≤ 2 секунды
- **NFR-003:** Обновление данных портфеля в фоновом режиме (не блокирует UI)

### 4.2 Безопасность
- **NFR-004:** Шифрование данных при передаче (HTTPS/TLS 1.3)
- **NFR-005:** Шифрование локальных данных (Keychain/Keystore)
- **NFR-006:** Автоматический logout после 30 минут неактивности
- **NFR-007:** Маскировка чувствительных данных на скриншотах
- **NFR-008:** Соответствие требованиям ЦБ РФ для финансовых приложений

### 4.3 Доступность
- **NFR-009:** Поддержка iOS 14+ и Android 8+
- **NFR-010:** Поддержка темной темы (dark mode)
- **NFR-011:** Базовая поддержка accessibility (VoiceOver, TalkBack)

### 4.4 Надежность
- **NFR-012:** Доступность приложения 99.5% времени
- **NFR-013:** Offline-режим с кэшированием последних данных
- **NFR-014:** Обработка потери соединения с информативным сообщением

### 4.5 Масштабируемость
- **NFR-015:** Архитектура поддерживает до 10,000 пользователей в начальной версии
- **NFR-016:** Возможность масштабирования до 100,000+ пользователей

## 5. Технические требования

### 5.1 Архитектура
Приложение должно иметь трехуровневую архитектуру:

**View Layer (Presentation):**
- React Native компоненты экранов
- UI компоненты из shadcn/ui
- Стилизация через Tailwind CSS
- Навигация (React Navigation)

**Domain Layer (Business Logic):**
- Сущности: User, Portfolio, Asset, Strategy, Message
- Use cases / Interactors
- State management (Redux Toolkit / Zustand)

**Adapters Layer (Infrastructure):**
- API клиент для бэкенда
- Repository паттерн для данных
- Маппинг данных (DTO ↔ Domain entities)

### 5.2 Технологический стек
| Компонент | Технология |
|-----------|------------|
| Framework | React Native |
| iOS Native | Swift / Objective-C |
| Android Native | Kotlin / Java |
| Styling | Tailwind CSS (NativeWind) |
| UI Kit | shadcn/ui (адаптированный для RN) |
| Navigation | React Navigation v6 |
| State Management | Redux Toolkit или Zustand |
| HTTP Client | Axios |
| Backend API | Node.js (REST или GraphQL) |
| Auth | JWT + Secure Storage |

### 5.3 API Requirements
- REST API с JSON
- WebSocket для чата (опционально для MVP можно polling)
- Аутентификация через JWT токены
- Refresh token механизм

## 6. Дизайн и UX

### 6.1 Дизайн-принципы
- **Минимализм** - чистый интерфейс без лишних элементов
- **Доверие** - профессиональный вид, строгая цветовая палитра
- **Простота** - интуитивная навигация, максимум 3 шага до любой функции

### 6.2 Навигация
- Bottom Tab Navigation с 4 вкладками:
  1. Главная / Портфель
  2. Продукты
  3. Чат
  4. Профиль

### 6.3 Цветовая схема
- Основной цвет: Темно-синий (#1E3A8A) - доверие, стабильность
- Акцентный цвет: Зеленый (#059669) - рост, успех
- Фон: Светло-серый / Белый
- Ошибки: Красный (#DC2626)

## 9. Выход за рамки (Out of Scope)

### 9.1 Версия MVP не включает:
- Торговые операции (покупка/продажа активов)
- Денежные переводы
- Интеграцию с банковскими системами
- Криптовалютные активы
- Робо-адвайзинг
- Социальные функции (рейтинги, отзывы)
- Web-версию приложения

### 9.2 Ограничения MVP:
- Только текстовый чат (без вложений, изображений, голосовых)
- Только один язык интерфейса (русский)
- Только базовая аутентификация (email/пароль)
- Нет оффлайн-режима для чата

## 10. Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Проблемы с интеграцией бэкенда | Средняя | Высокое | Ранняя интеграция, моковые данные |
| Низкая производительность на старых устройствах | Средняя | Среднее | Тестирование на реальных устройствах |
| Задержки в разработке UI | Низкая | Среднее | Использование готовых UI компонентов |
| Сложности с безопасностью данных | Средняя | Высокое | Аудит безопасности перед запуском |
