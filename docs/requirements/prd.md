# Product Requirements Document

## 1. Обзор

<b>Описание:</b> Моя идея следующая. Я хочу создать мобильное приложение на базе технологии react native для платформ ios и Android. Приложение создаётся для клиентов управляющей компании. Управляющая компания - это компания, которая занимается управлением активов клиентов. Через приложение клиент должен иметь возможность делать три базовые вещи. Клиент должен иметь возможность просматривать витрину продуктов компании - различные стратегии доверительного управления и продукт по индивидуальному доверительному управлению. Также клиент на отдельном экране должен иметь возможность просматривать свой текущий портфель активов под управлением. Третья главная фича - экран с in-app чатом со своим персональным менеджером, управляющим активами.

<b>Статус:</b> Draft
<b>Дата:</b> 2026-04-07

## 2. Целевая аудитория

Целевая аудитория - состоятельные инвесторы, которые передают свои активы или часть своих активов в доверительное управление управляющей компании.

## 3. Ключевые функции

Обязательные ключевые функции - просмотр витрины продуктов, просмотр текущего портфеля активов под управлением, чат с персональным менеджером и управляющим.

## 4. Ограничения

Ограничения по стеку. Приложение для платформ ios и Android на базе react native. Приложение должно иметь архитектуру, позволяющую подключать его к бэкенду, составляющему данные. В приложении должно быть три архитектурных слоя: view с описанием экранов, domain с описанием базовых сущностей и бизнес-логикой, adapters для интеграции domain слоя с бэкендом.

## 5. Технический стек

- Frontend: React Native
- Backend: Node.js
- Styling: Tailwind CSS
- UI Kit: shadcn/ui

## 6. Успешные критерии

- MVP готов к тестированию
- Основные фичи работают
- Код протестирован
