# Product Requirements Document (PRD)

## 1. Обзор продукта

### 1.1 Описание продукта
Мобильное приложение для клиентов управляющей компании, позволяющее просматривать инвестиционные продукты, отслеживать портфель активов и общаться с персональным менеджером через in-app чат.

### 1.2 Статус
Draft

### 1.3 Дата создания
2026-04-07

### 1.4 Версия документа
1.0

---

## 2. Цели и задачи

### 2.1 Бизнес-цели
- Обеспечить клиентам удобный мобильный доступ к информации об инвестиционных продуктах и портфеле
- Улучшить коммуникацию между клиентами и персональными менеджерами
- Повысить удовлетворенность клиентов и их вовлеченность
- Снизить нагрузку на операционные каналы связи (телефон, email)

### 2.2 Продуктовые цели
- Создать интуитивно понятный интерфейс для просмотра инвестиционных продуктов
- Обеспечить прозрачность портфеля активов в реальном времени
- Организовать эффективный канал коммуникации через in-app чат

### 2.3 Ключевые метрики успеха (KPI)
- Количество активных пользователей (DAU/MAU)
- Время, проведенное в приложении
- Количество сообщений в чате
- Retention rate на 7/30/90 дней
- App Store и Google Play рейтинги

---

## 3. Целевая аудитория и персоны

### 3.1 Целевая аудитория
Состоятельные инвесторы, передающие свои активы или часть активов в доверительное управление управляющей компании.

### 3.2 Детальные персоны

#### Персона 1: Опытный инвестор

**Имя:** Александр
**Возраст:** 45-55 лет
**Профессия:** Владелец бизнеса / топ-менеджер
**Доход:** Высокий

**Характеристики:**
- Имеет опыт инвестирования через доверительное управление
- Интересуется рынками и стратегиями управления
- Ценит прозрачность и оперативность информации
- Предпочитает минимальные касания с менеджерами для рутинных вопросов
- Технически грамотен, активно использует смартфон

**Боли:**
- Отсутствие быстрого доступа к информации о портфеле
- Долгое время ответа менеджеров на простые вопросы
- Необходимость звонить или писать email для получения базовой информации

**Потребности:**
- Просмотр текущего портфеля в любой момент
- Быстрый доступ к информации о продуктах и стратегиях
- Возможность быстро связаться с менеджером по важным вопросам

#### Персона 2: Консервативный клиент

**Имя:** Елена
**Возраст:** 50-60 лет
**Профессия:** На пенсии / рантье
**Доход:** Высокий

**Характеристики:**
- Передала активы в управление для сохранения капитала
- Менее опытна в инвестиционных вопросах
- Ценит личный контакт и внимание менеджера
- Нуждается в объяснении сложных финансовых концепций
- Использует смартфон для базовых задач

**Боли:**
- Сложность понимания инвестиционных стратегий
- Тревога о состоянии портфеля
- Необходимость личного общения для решения вопросов

**Потребности:**
- Простое и понятное отображение портфеля
- Чат для общения с персональным менеджером
- Понятное описание продуктов и стратегий

## 5. Нефункциональные требования

### 5.1 Производительность
- Время запуска приложения: не более 3 секунд
- Время загрузки экранов: не более 2 секунд
- Плавная анимация: 60 FPS
- Поддержка устройств с 2GB RAM и выше

### 5.2 Безопасность
- Шифрование данных при передаче (HTTPS/TLS 1.3)
- Шифрование локального хранилища
- Соответствие требованиям регуляторов (ФСФР, ЦБ РФ)
- Логирование критических операций
- Защита от MITM атак

### 5.3 Надежность
- Доступность приложения: 99.5% uptime
- Автоматическое восстановление при сбоях
- Обработка потери соединения с сервером
- Graceful degradation при ошибках API

### 5.4 Масштабируемость
- Поддержка до 10,000 активных пользователей
- Возможность горизонтального масштабирования бэкенда
- Оптимизация API запросов

### 5.5 Совместимость
- iOS 13.0 и выше
- Android 8.0 (API level 26) и выше
- Поддержка различных размеров экранов (от 4.7 до 7 дюймов)
- Поддержка планшетов (iPad, Android tablets)

### 5.6 Локализация
- Русский язык (основной)
- Возможность добавления других языков (архитектурная поддержка)

## 6. Технический стек

### 6.1 Frontend (Mobile App)
- **Framework:** React Native
- **Стилизация:** Tailwind CSS (через NativeWind или аналог)
- **UI Kit:** shadcn/ui (адаптированный для React Native)
- **State Management:** Redux Toolkit / Zustand
- **Navigation:** React Navigation
- **Networking:** Axios / React Query

### 6.2 Backend
- **Runtime:** Node.js
- **Framework:** Express / NestJS
- **Database:** PostgreSQL
- **Cache:** Redis
- **Message Queue:** Redis / RabbitMQ (для чата)

### 6.3 Архитектура приложения

Приложение следует принципам Clean Architecture с тремя основными слоями:

#### View Layer (Presentation)
- Компоненты экранов (Screens)
- UI компоненты (Components)
- Навигация (Navigation)
- Стили (Styles)

#### Domain Layer (Business Logic)
- Сущности (Entities): User, Portfolio, Product, Message
- Use Cases / Interactors
- Бизнес-логика и правила валидации
- Интерфейсы репозиториев

#### Adapters Layer (Infrastructure)
- API клиенты для бэкенда
- Реализация репозиториев
- Мапперы данных
- Локальное хранилище (AsyncStorage, SecureStore)

## 7. Интеграции

### 7.1 Бэкенд управляющей компании
- REST API для получения данных о продуктах
- REST API для получения данных о портфеле
- WebSocket для real-time обновлений портфеля (опционально)

### 7.2 Система чата
- REST API для отправки/получения сообщений
- WebSocket для real-time доставки сообщений
- Push-уведомления (Firebase Cloud Messaging, APNs)

### 7.3 Внешние сервисы
- Firebase Cloud Messaging (Android push)
- Apple Push Notification Service (iOS push)
- Сервис аналитики (Google Analytics / Amplitude)
- Crashlytics для отслеживания ошибок

## 8. Ограничения и предположения

### 8.1 Ограничения
- MVP должен быть готов к тестированию в кратчайшие сроки
- Бюджет и сроки определяются отдельно
- Команда разработки ограничена (solo founder scenario)

### 8.2 Предположения
- Бэкенд API уже существует или разрабатывается параллельно
- Есть доступ к тестовым данным о продуктах и портфелях
- Есть персональные менеджеры для назначения клиентам
- Процедура KYC (Know Your Customer) выполняется вне приложения

---

## 9. Элементы вне объема (Out of Scope)

### 9.1 Не включено в MVP
- Открытие новых счетов через приложение
- Ввод/вывод средств через приложение
- Подача заявок на инвестиционные продукты
- Торговые операции
- Интеграция с банковскими системами
- Мультиязычность (кроме русского)
- Виджет для домашнего экрана

### 9.2 Возможное развитие в будущем
- Биометрическая аутентификация для транзакций
- Интеграция с внешними брокерами
- AI-ассистент для инвестиционных рекомендаций
- Социальные функции (рейтинг стратегий)
- Геймификация и программы лояльности

## 10. Критерии успешности MVP

### 10.1 Функциональные критерии
- ✅ Пользователь может авторизоваться
- ✅ Пользователь может просматривать витрину продуктов
- ✅ Пользователь может просматривать свой портфель
- ✅ Пользователь может отправлять и получать сообщения в чате
- ✅ Push-уведомления о новых сообщениях работают
- ✅ Приложение работает на iOS и Android

### 10.2 Технические критерии
- ✅ Код покрыт unit-тестами (минимум 60% coverage)
- ✅ Приложение прошло code review
- ✅ Нет критических багов
- ✅ Производительность соответствует требованиям

### 10.3 Продуктовые критерии
- ✅ MVP протестирован с реальными пользователями (бета-тест)
- ✅ Собрана обратная связь от первых пользователей
- ✅ App Store и Google Play рейтинги не ниже 4.0
