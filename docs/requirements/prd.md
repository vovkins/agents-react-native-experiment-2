## Приоритизация MoSCoW

- **M (Must Have)** — Обязательно для MVP
- **S (Should Have)** — Важно, но можно реализовать после MVP
- **C (Could Have)** — Желательно, если есть ресурсы
- **W (Won't Have)** — Не включено в текущую версию

## MVP Эпики (Must Have)

### 1. Authentication & Security System
**GitHub Issue:** [#59](https://github.com/vovkins/agents-react-native-experiment-2/issues/59)
**Приоритет:** Must Have (MVP)
**Статус:** Open

**Описание:**
Реализация системы аутентификации и безопасности для мобильного приложения управляющей компании.

**Ключевые фичи:**
- Вход через логин/пароль (email/телефон)
- Автоматический выход из сессии через 30 минут
- Восстановление пароля через email/SMS

**Acceptance Criteria:**
- Пользователь может авторизоваться через email/телефон и пароль
- Автоматический logout после 30 минут неактивности
- Функция восстановления пароля работает корректно
- Работает на iOS и Android

---

### 2. Product Showcase Screen
**GitHub Issue:** [#60](https://github.com/vovkins/agents-react-native-experiment-2/issues/60)
**Приоритет:** Must Have (MVP)
**Статус:** Open

**Описание:**
Реализация экрана витрины инвестиционных продуктов управляющей компании с отображением стратегий доверительного управления и продукта ИДУ.

**Ключевые фичи:**
- Список стратегий доверительного управления
- Карточки продуктов с ключевыми параметрами
- Детальная страница продукта
- Продукт индивидуального доверительного управления (ИДУ)

**Acceptance Criteria:**
- Отображение всех доступных инвестиционных стратегий
- Карточки продуктов содержат название, описание, параметры
- Детальная страница показывает полную информацию о стратегии
- Pull-to-refresh обновляет список продуктов
- Корректная обработка ошибок сети

---

### 3. Portfolio Management Screen
**GitHub Issue:** [#61](https://github.com/vovkins/agents-react-native-experiment-2/issues/61)
**Приоритет:** Must Have (MVP)
**Статус:** Open

**Описание:**
Реализация экрана портфеля активов клиента с отображением текущей стоимости, структуры портфеля, списка позиций и доходности.

**Ключевые фичи:**
- Общая стоимость портфеля в валюте
- Структура портфеля по классам активов
- Список позиций с текущей стоимостью
- Доходность портфеля (абсолютная и относительная)

**Acceptance Criteria:**
- Отображение общей стоимости портфеля с индикатором изменения
- Визуализация структуры портфеля (pie chart)
- Детальный список всех активов
- Корректный расчет доходности
- Pull-to-refresh обновляет данные

---

### 4. In-app Chat with Manager
**GitHub Issue:** [#62](https://github.com/vovkins/agents-react-native-experiment-2/issues/62)
**Приоритет:** Must Have (MVP)
**Статус:** Open

**Описание:**
Реализация in-app чата для коммуникации клиента с персональным менеджером управляющей компании.

**Ключевые фичи:**
- Отправка текстовых сообщений
- История сообщений
- Push-уведомления о новых сообщениях
- Информация о менеджере (имя, фото, должность)
- Real-time обновления через WebSocket

**Acceptance Criteria:**
- Пользователь может отправлять и получать текстовые сообщения
- История сообщений сохраняется и загружается
- Push-уведомления работают даже когда приложение закрыто
- Информация о менеджере отображается в шапке чата
- Real-time доставка сообщений через WebSocket

### 5. Core Application Infrastructure
**GitHub Issue:** [#63](https://github.com/vovkins/agents-react-native-experiment-2/issues/63)
**Приоритет:** Must Have (MVP)
**Статус:** Open

**Описание:**
Реализация базовой инфраструктуры мобильного приложения: навигация, профиль пользователя, настройки, обработка ошибок.

**Ключевые фичи:**
- Навигация по приложению (Tab Bar)
- Профиль пользователя
- Настройки приложения
- Pull-to-refresh на всех экранах
- Обработка ошибок

**Acceptance Criteria:**
- Нижняя панель навигации с 4 разделами (Продукты, Портфель, Чат, Профиль)
- Экран профиля с информацией о клиенте
- Экран настроек с управлением уведомлениями
- Понятные сообщения об ошибках на русском языке
- Pull-to-refresh работает на всех экранах с данными

## Метрики успеха MVP

### Функциональные метрики
- ✅ Все 5 эпиков реализованы
- ✅ Все acceptance criteria выполнены
- ✅ Приложение работает на iOS и Android
- ✅ Unit-тест coverage > 60%

### Технические метрики
- Время запуска приложения < 3 секунд
- Время загрузки экранов < 2 секунд
- Нет критических багов
- Code review пройден для всех эпиков

### Продуктовые метрики
- MVP протестирован с реальными пользователями
- Собрана обратная связь
- App Store / Google Play рейтинги ≥ 4.0

## Зависимости и риски

### Критические зависимости
- Бэкенд API должен быть готов к началу разработки
- WebSocket сервер для чата должен быть настроен
- Firebase/APNs для push-уведомлений должен быть настроен
- UI Kit (shadcn/ui) должен быть адаптирован для React Native

### Риски
| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Отсутствие готового бэкенда | Средняя | Высокое | Разработка mock-сервера для разработки и тестирования |
| Сложности с WebSocket интеграцией | Средняя | Среднее | Использование готовых решений (Socket.io) |
| Задержки с настройкой push-уведомлений | Низкая | Среднее | Ранняя интеграция Firebase/APNs |
