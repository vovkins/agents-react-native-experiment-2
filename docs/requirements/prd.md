# Product Requirements Document (PRD)

## 1. Обзор продукта

### 1.1 Описание
Мобильное приложение для клиентов управляющей компании, предоставляющее доступ к управлению активами через смартфон. Приложение позволяет клиентам просматривать инвестиционные продукты, отслеживать портфель активов и общаться с персональным менеджером.

**Статус:** Draft  
**Дата создания:** 2026-04-03  
**Версия:** 1.0

### 1.2 Проблема
Состоятельные инвесторы, передающие активы в доверительное управление, испытывают следующие трудности:
- Отсутствие удобного мобильного доступа к информации о своих инвестициях
- Невозможность быстро просмотреть доступные стратегии управления
- Сложности в оперативной коммуникации с персональным менеджером
- Необходимость использовать веб-портал или телефон для базовых операций

### 1.3 Решение
Мобильное приложение на React Native, которое:
- Обеспечивает мгновенный доступ к витрине инвестиционных продуктов
- Позволяет мониторить портфель активов в реальном времени
- Предоставляет прямой канал связи с персональным менеджером через in-app чат

### 1.4 Цели продукта

**Бизнес-цели:**
- Улучшить клиентский опыт и удовлетворенность клиентов
- Снизить нагрузку на операционный отдел за счет самообслуживания
- Повысить вовлеченность клиентов в управление активами
- Создать конкурентное преимущество на рынке доверительного управления

**Продуктовые цели:**
- Запустить MVP в течение 3-4 месяцев
- Обеспечить базовую функциональность для 3 ключевых сценариев использования
- Поддержать iOS и Android платформы

### 1.5 Success Metrics (Критерии успеха)
- ✅ MVP готов к тестированию
- ✅ Основные фичи работают корректно
- ✅ Код протестирован (unit tests, integration tests)
- 📊 Средняя оценка в App Store/Google Play ≥ 4.0 (после запуска)
- 📊 Monthly Active Users (MAU) ≥ 50% от базы клиентов (после запуска)
- 📊 Retention rate Day 30 ≥ 40%

## 2. Целевая аудитория

### 2.1 Основная аудитория
**Состоятельные инвесторы** — физические лица с высоким уровнем достатка, которые передают свои активы или их часть в доверительное управление компании.

**Характеристики:**
- Возраст: 35-65 лет
- Доход: высокий (upper-middle to high income)
- Образование: высшее, часто финансовое или техническое
- Инвестиционный опыт: от 5 лет
- Активы под управлением: от 1 млн рублей

### 2.2 User Personas

#### Persona 1: "Опытный инвестор" — Александр, 45 лет

**Профиль:**
- Топ-менеджер крупной компании
- Инвестирует 5+ лет
- Активы под управлением: 10-15 млн рублей
- Использует iOS устройства

**Цели:**
- Быстро проверять состояние портфеля
- Изучать новые инвестиционные стратегии
- Оперативно связываться с менеджером по вопросам ребалансировки

**Боли:**
- Нет времени заходить в веб-кабинет
- Хочет видеть ключевую информацию одним взглядом
- Необходимость быстро принимать решения

**Сценарии использования:**
- Просмотр портфеля утром за кофе (ежедневно)
- Изучение новых стратегий перед встречей с менеджером (раз в месяц)
- Чат с менеджером при возникновении вопросов (по необходимости)

---

#### Persona 2: "Пассивный инвестор" — Елена, 52 года

**Профиль:**
- Владелец бизнеса
- Инвестирует 3+ года
- Активы под управлением: 5-8 млн рублей
- Использует Android устройства

**Цели:**
- Контролировать рост капитала
- Получать обновления от менеджера
- Минимизировать время на управление инвестициями

**Боли:**
- Не разбирается в сложных финансовых терминах
- Хочет простую и понятную информацию
- Боится пропустить важные уведомления

**Сценарии использования:**
- Просмотр портфеля раз в неделю
- Чтение сообщений от менеджера
- Просмотр годовой доходности

## 3. Функциональные требования

### 3.1 Приоритизация MoSCoW

**MUST HAVE (Обязательно для MVP):**

#### F1. Аутентификация и безопасность
- **F1.1** Вход в приложение по логину/паролю
- **F1.2** Биометрическая аутентификация (Face ID/Touch ID) для повторных входов
- **F1.3** Автоматический выход после периода неактивности (15 минут)
- **F1.4** Защищенное хранение токенов авторизации

#### F2. Витрина продуктов (Product Showcase)
- **F2.1** Просмотр списка доступных стратегий доверительного управления
- **F2.2** Детальная информация по каждой стратегии (описание, доходность, риск, минимальная сумма)
- **F2.3** Просмотр продукта индивидуального доверительного управления
- **F2.4** Фильтрация и сортировка продуктов

#### F3. Портфель активов (Portfolio View)
- **F3.1** Просмотр текущего портфеля активов под управлением
- **F3.2** Отображение структуры портфеля (круговая диаграмма)
- **F3.3** Просмотр доходности портфеля (за период, с начала года)
- **F3.4** Детализация по каждому активу в портфеле

#### F4. Чат с менеджером (In-App Chat)
- **F4.1** Отправка текстовых сообщений персональному менеджеру
- **F4.2** Получение сообщений от менеджера в реальном времени
- **F4.3** История переписки
- **F4.4** Push-уведомления о новых сообщениях

**SHOULD HAVE (Важно, но не критично для MVP):**

#### F5. Уведомления
- **F5.1** Push-уведомления о важных событиях портфеля
- **F5.2** Настройка уведомлений (вкл/выкл)

#### F6. Профиль пользователя
- **F6.1** Просмотр личных данных
- **F6.2** Контактная информация менеджера
- **F6.3** Настройки приложения

#### F7. Документы
- **F7.1** Просмотр отчетов по портфелю (PDF)
- **F7.2** Просмотр договоров

**COULD HAVE (Желательно в будущих версиях):**

#### F8. Расширенная аналитика
- **F8.1** Графики доходности по периодам
- **F8.2** Сравнение с бенчмарками
- **F8.3** Прогнозы и рекомендации

#### F9. Дополнительные функции чата
- **F9.1** Отправка изображений и документов
- **F9.2** Голосовые сообщения

#### F10. Биометрическая аутентификация при первом входе
- **F10.1** Возможность использовать биометрию сразу после регистрации

**WON'T HAVE (Не планируется):**

- Торговые операции (покупка/продажа активов через приложение)
- Вывод средств
- Открытие новых счетов
- Интеграция с внешними биржами
- Социальные функции (форумы, комментарии)

## 4. Нефункциональные требования

### 4.1 Производительность
- **NFR1** Время запуска приложения ≤ 3 секунды
- **NFR2** Время загрузки экранов ≤ 2 секунды
- **NFR3** Поддержка offline-режима для просмотренных данных (кэширование)
- **NFR4** Оптимизация потребления батареи

### 4.2 Безопасность
- **NFR5** Шифрование данных при передаче (HTTPS/TLS 1.2+)
- **NFR6** Шифрование локальных данных (iOS Keychain, Android Keystore)
- **NFR7** Соответствие требованиям регулятора (ЦБ РФ)
- **NFR8** Защита от MITM атак (Certificate Pinning)
- **NFR9** Аудит логов действий пользователя

### 4.3 Надежность
- **NFR10** Доступность 99.5%
- **NFR11** Graceful degradation при потере соединения
- **NFR12** Автоматическое восстановление сессии

### 4.4 Масштабируемость
- **NFR13** Поддержка до 10,000 одновременных пользователей
- **NFR14** Архитектура, позволяющая горизонтальное масштабирование

### 4.5 Совместимость
- **NFR15** iOS 13.0+
- **NFR16** Android 8.0+ (API level 26)
- **NFR17** Поддержка планшетов (adaptive layout)

### 4.6 Удобство использования (Usability)
- **NFR18** Интуитивно понятный интерфейс (без обучения)
- **NFR19** Поддержка темной темы (Dark Mode)
- **NFR20** Доступность (Accessibility) — VoiceOver/TalkBack
- **NFR21** Локализация: русский язык

## 5. Технические требования

### 5.1 Технологический стек

**Frontend:**
- Framework: React Native (Expo)
- Language: TypeScript
- Styling: Tailwind CSS (twrnc или NativeWind)
- UI Components: shadcn/ui (адаптированный для React Native)
- State Management: Zustand или Redux Toolkit
- Navigation: React Navigation v6

**Backend:**
- Runtime: Node.js
- API: RESTful API или GraphQL
- Database: PostgreSQL
- Cache: Redis
- Message Queue: RabbitMQ (для чата)

**Infrastructure:**
- CI/CD: GitHub Actions
- Monitoring: Sentry (ошибки), Firebase Analytics
- Push Notifications: Firebase Cloud Messaging

### 5.2 Архитектура

Трехуровневая архитектура:

```
┌─────────────────────────────────────┐
│           VIEW LAYER                │
│  (UI Components, Screens)           │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│          DOMAIN LAYER               │
│  (Entities, Business Logic,         │
│   Use Cases)                        │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│         ADAPTERS LAYER              │
│  (API Clients, Storage,             │
│   External Services)                │
└─────────────────────────────────────┘
```

**View Layer:**
- React Native компоненты
- Экраны приложения
- Навигация
- UI Kit компоненты

**Domain Layer:**
- Бизнес-сущности (Portfolio, Product, Message, User)
- Use Cases (GetPortfolio, GetProducts, SendMessage)
- Business Logic (расчет доходности, валидация)

**Adapters Layer:**
- API Client для коммуникации с бэкендом
- Local Storage (AsyncStorage, SQLite)
- Push Notifications Service
- Analytics Service

### 5.3 API Requirements
- RESTful API с JSON
- Authentication: JWT tokens
- Real-time: WebSockets для чата
- Versioning: /api/v1/

## 10. Acceptance Criteria (Критерии приемки)

### 10.1 Общие критерии
- ✅ Приложение запускается на iOS и Android
- ✅ Все MUST HAVE функции реализованы
- ✅ Unit test coverage ≥ 70%
- ✅ Integration tests для ключевых сценариев
- ✅ Нет критических багов
- ✅ Performance requirements соблюдены

### 10.2 Критерии по функциям

**Аутентификация:**
- Пользователь может войти с валидными credentials
- Биометрия работает корректно
- Сессия автоматически завершается после таймаута

**Витрина продуктов:**
- Отображаются все доступные продукты
- Детальная информация корректна
- Фильтрация работает

**Портфель:**
- Данные портфеля актуальны
- Диаграммы отображаются корректно
- Расчеты доходности верны

**Чат:**
- Сообщения доставляются в реальном времени
- Push-уведомления приходят
- История сохраняется
