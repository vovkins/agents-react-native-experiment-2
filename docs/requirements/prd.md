# Product Backlog - Prioritized

Данный документ содержит приоритизированный список всех эпиков и фич для мобильного приложения управляющей компании.

## Приоритизация

### Шкала приоритетов

- **P0 (Critical)** — Критически важно, обязательно для MVP
- **P1 (High)** — Высокий приоритет, должно быть в первой версии
- **P2 (Medium)** — Средний приоритет, желательно реализовать
- **P3 (Low)** — Низкий приоритет, nice to have

---

## P0 - Critical (MVP Must Have)

### 1. Core Application Infrastructure
**GitHub Issue:** [#63](https://github.com/vovkins/agents-react-native-experiment-2/issues/63)
**Приоритет:** P0 - Critical (MVP)
**Статус:** Open

**Обоснование приоритета:**
Базовая инфраструктура является фундаментом для всех остальных функций. Должна быть реализована первой.

**Ключевые фичи:**
- Навигация по приложению (Tab Bar)
- Профиль пользователя
- Настройки приложения
- Pull-to-refresh
- Обработка ошибок

**Зависимости:** Нет
**Рекомендуемый спринт:** Спринт 1

---

### 2. Authentication & Security System
**GitHub Issue:** [#59](https://github.com/vovkins/agents-react-native-experiment-2/issues/59)
**Приоритет:** P0 - Critical (MVP)
**Статус:** Open

**Обоснование приоритета:**
Аутентификация обязательна для доступа к персональным данным клиента. Критически важна для безопасности финансового приложения.

**Ключевые фичи:**
- Вход через логин/пароль (email/телефон)
- Автоматический выход из сессии через 30 минут
- Восстановление пароля через email/SMS

**Зависимости:** Core Application Infrastructure
**Рекомендуемый спринт:** Спринт 2

---

### 3. Product Showcase Screen
**GitHub Issue:** [#60](https://github.com/vovkins/agents-react-native-experiment-2/issues/60)
**Приоритет:** P0 - Critical (MVP)
**Статус:** Open

**Обоснование приоритета:**
Витрина продуктов - одна из трех ключевых функций приложения. Позволяет клиентам знакомиться с инвестиционными возможностями.

**Ключевые фичи:**
- Список стратегий доверительного управления
- Карточки продуктов с ключевыми параметрами
- Детальная страница продукта
- Продукт индивидуального доверительного управления (ИДУ)

**Зависимости:** Authentication & Security System, Backend API
**Рекомендуемый спринт:** Спринт 3

---

### 4. Portfolio Management Screen
**GitHub Issue:** [#61](https://github.com/vovkins/agents-react-native-experiment-2/issues/61)
**Приоритет:** P0 - Critical (MVP)
**Статус:** Open

**Обоснование приоритета:**
Просмотр портфеля - одна из трех ключевых функций приложения. Клиенты должны видеть свои активы в реальном времени.

**Ключевые фичи:**
- Общая стоимость портфеля в валюте
- Структура портфеля по классам активов
- Список позиций с текущей стоимостью
- Доходность портфеля (абсолютная и относительная)

**Зависимости:** Authentication & Security System, Backend API
**Рекомендуемый спринт:** Спринт 3

---

### 5. In-app Chat with Manager
**GitHub Issue:** [#62](https://github.com/vovkins/agents-react-native-experiment-2/issues/62)
**Приоритет:** P0 - Critical (MVP)
**Статус:** Open

**Обоснование приоритета:**
In-app чат - одна из трех ключевых функций приложения. Обеспечивает персональную коммуникацию с менеджером.

**Ключевые фичи:**
- Отправка текстовых сообщений
- История сообщений
- Push-уведомления о новых сообщениях
- Информация о менеджере (имя, фото, должность)
- Real-time обновления через WebSocket

**Зависимости:** Authentication & Security System, WebSocket Server, Push Notifications
**Рекомендуемый спринт:** Спринт 4

## Метрики успеха MVP

### Функциональные метрики
- ✅ Все 5 P0 эпиков реализованы
- ✅ Все acceptance criteria выполнены
- ✅ Приложение работает на iOS и Android
- ✅ Unit-тест coverage > 60%

### Технические метрики
- Время запуска приложения < 3 секунд
- Время загрузки экранов < 2 секунд
- Нет критических багов
- Code review пройден для всех эпиков

### Продуктовые метрики
- MVP протестирован с реальными пользователями
- Собрана обратная связь
- App Store / Google Play рейтинги ≥ 4.0
- Retention rate на 7 дней ≥ 40%

## Зависимости и блокеры

### Критические зависимости для MVP
| Зависимость | Эпик | Статус | Ответственный |
|-------------|------|--------|---------------|
| React Native проект настроен | Core Infrastructure | Required | Dev Team |
| Backend API готов | Auth, Products, Portfolio | Required | Backend Team |
| WebSocket сервер настроен | Chat | Required | Backend Team |
| Firebase проект настроен | Chat (Push) | Required | DevOps |
| APNs сертификаты | Chat (Push iOS) | Required | DevOps |
| UI Kit адаптирован | All screens | Required | UI/UX Team |

## Риски и митигация

| Риск | Вероятность | Влияние | Эпик | Митигация |
|------|-------------|---------|------|-----------|
| Отсутствие готового бэкенда | Средняя | Высокое | All | Разработка mock-сервера для разработки |
| Сложности с WebSocket интеграцией | Средняя | Среднее | Chat | Использование готовых решений (Socket.io) |
| Задержки с настройкой push-уведомлений | Низкая | Среднее | Chat | Ранняя интеграция Firebase/APNs |
| Проблемы с производительностью | Низкая | Среднее | All | Оптимизация и тестирование на целевых устройствах |
| Задержки с App Store/Google Play | Низкая | Среднее | Release | Раннее изучение требований |
