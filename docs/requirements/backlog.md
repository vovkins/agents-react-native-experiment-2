## Legend

- **P0** - Must Have (критично для MVP) - Блокирует запуск продукта
- **P1** - Should Have (важно, но не блокирует MVP) - Важно для конкурентоспособности
- **P2** - Could Have (желательно) - Улучшает UX, но не критично
- **P3** - Won't Have (не в текущей версии) - Отложено до будущих релизов

| Milestone | Описание | Сроки | Количество задач |

## Priority Definitions

### P0 - Must Have (MVP Critical)
**Определение:** Критически важно для запуска MVP. Без этих функций продукт не может быть выпущен.

**Критерии:**
- Блокирует основные пользовательские сценарии
- Требуется для соответствия regulation (финансовый сектор)
- Является ключевым дифференциатором продукта

**Количество:** 25 задач, 116 SP

---

### P1 - Should Have (High Priority)
**Определение:** Важные функции, которые значительно улучшают продукт, но не блокируют MVP запуск.

**Критерии:**
- Повышает конкурентоспособность
- Улучшает UX и retention
- Часто запрашивается пользователями

**Количество:** 14 задач, 52 SP

---

### P2 - Could Have (Medium Priority)
**Определение:** Желательные функции, которые улучшают продукт, но не критичны.

**Критерии:**
- Nice to have
- Может быть отложено
- Небольшое влияние на метрики

**Количество:** 1 задача, 5 SP

---

### P3 - Won't Have (Low Priority)
**Определение:** Функции, которые отложены до будущих версий.

**Критерии:**
- Требует значительных ресурсов
- Не соответствует текущим целям
- Может быть пересмотрено в будущем

**Количество:** 0 задач в текущем бэклоге
